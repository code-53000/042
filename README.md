
# 🔧 Gunpla 涂装进度册 | 高达模型工作台

专为模型涂装爱好者打造的个人涂装进度追踪工具。记录每台模型的比例、系列、涂装阶段、使用漆料品牌与调色配方，通过多维度筛选快速定位半成品与完成品。

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 📦 **模型档案** | 记录模型名称、系列（MG/HG/RG/PG等）、比例、购入/开工/完工日期、备注 |
| 🎨 **涂装步骤时间线** | 9 个标准涂装阶段（素组→打磨→水补→分色→细节→旧化→水贴→消光→完工），可切换状态 |
| 🧪 **漆料与配方** | 关联每步使用的漆料（品牌+色号+颜色名），支持自定义调色配方和比例记录 |
| 🔍 **多维筛选** | 按系列、涂装阶段、完成状态、完成度区间、关键词搜索过滤 |
| 📊 **统计仪表盘** | 完成率环图、系列占比饼图、阶段进度汇总条、月度开工/完工趋势 |
| 💾 **本地存储** | 纯前端方案，数据完全存储于浏览器 IndexedDB，无需后端服务 |

## 🏗️ 技术架构

四层分离设计，各模块职责独立，方便后续扩展漆料库存、成品摄影等功能：

```
src/
├── db/              ← IndexedDB 数据访问层（纯数据操作，无业务逻辑）
├── services/        ← 业务逻辑层（进度计算、筛选逻辑、状态联动）
├── stores/          ← Zustand 状态层（React 组件订阅用）
├── components/      ← UI 组件（按模型/涂装/统计/通用分组）
├── pages/           ← 页面组件
├── types/           ← TypeScript 类型定义
└── utils/           ← 工具函数与示例数据
```

## 📖 使用指南

### 1. 新建模型档案
1. 点击右上角 **「新建模型」** 按钮
2. 填写名称、选择系列（MG/HG/RG等）和比例
3. 选择一个可爱的 emoji 图标（可选）
4. 填写购入日期、开工日期、备注
5. 保存后自动生成 9 个默认涂装步骤

### 2. 记录涂装步骤
1. 在「模型总览」点击卡片进入详情页
2. 点击任意涂装步骤卡片展开编辑面板
3. 点击步骤右上角的 **「切换」** 按钮循环切换状态：`待开始 → 进行中 → 已完成 → 跳过`
4. 点击 **「添加漆料」** 记录该步骤使用的漆料：
   - 选择品牌（郡士/田宫/盖亚/匠域/AV 等）
   - 填写色号和颜色名
   - 勾选「自定义调色」可记录配方比例（如：`GA-030 + GA-020 3:1`）

### 3. 筛选 & 统计
- **总览页**：使用顶部筛选栏按系列/状态/阶段/完成度过滤
- **统计仪表盘**：导航栏点击「统计仪表盘」查看完整数据可视化

## 🔌 可扩展方向

当前架构已预留扩展位，新增功能不会破坏现有结构：

| 扩展模块 | 建议路径 |
|----------|----------|
| 🎨 漆料库存管理 | `src/services/PaintInventoryService.ts` + 新建页面 + `db/paintInventory.ts` |

| 🏷️ 标签系统 | 在 `Model` 类型新增 `tags: string[]` 字段，筛选栏新增标签维度 |
| 📅 制作日志 | 新增 `build_logs` 对象仓，支持按日期记录制作笔记 |
